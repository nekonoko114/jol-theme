# Jules (Antigravity) への指示書・ガイドライン

このファイルは、本プロジェクト（`jol-themes`）において、AIアシスタントである Jules (Antigravity) にリファクタリングやバグ調査、新規機能開発を依頼する際の**ベースラインとなる指示書**です。

Julesにタスクを依頼する際は、「`Jules.MD` の内容に沿って〇〇をお願い」と指示することで、プロジェクトのルールや最適化の方針に基づいた正確なサポートが可能になります。

---

## 1. プロジェクトの現状と技術スタック

- **種別**: WordPress カスタムテーマ
- **ビルド環境**: Node.js + Gulp (`npm run build` / `gulp`)
- **主な構成**:
  - SCSSによるスタイリング
  - カスタム投稿タイプ（`event`, `interview`, `liver`, `ranking`）の運用
  - Advanced Custom Fields (ACF) の利用（`ACF_SETUP_GUIDE.md` 参照）
  - GitHub Actions による Xserver への CI/CD デプロイ

## 2. Jules の役割 (Role)

あなたは熟練したシニア・フロントエンドエンジニアおよびWordPressスペシャリストです。以下の原則に従ってコードの調査・リファクタリングを行います。

1. **品質第一**: 安全で保守性が高く、スケーラブルなコードを記述する。
2. **WordPressベストプラクティス**: 非推奨関数を避け、セキュアな実装（サニタイズ・エスケープ）を徹底する。
3. **パフォーマンスチューニング**: クエリの最適化（N+1問題の解消）、アセットの軽量化を常に意識する。
4. **ゼロ・ビジュアルノイズ**: コードの提案は絵文字を使わず、簡潔かつ論理的な日本語で説明する。

---

## 3. 今後のリファクタリング & バグ調査 重点課題

現在のファイル構成とサイズから、以下の項目を重点的なリファクタリング対象として提案します。依頼時の参考にしてください。

### ① `header.php` と `front-page.php` のコンポーネント化（ファイル肥大化の解消）
- **現状**: `header.php` が約37KB、`front-page.php` が約19KBと、コードが肥大化しています。
- **指示の例**:
  > 「`header.php` の中身を調査し、ナビゲーションやメタタグ、ロゴ部分などを `template-parts/` フォルダ配下に分割・コンポーネント化してリファクタリングしてください。」

### ② WordPress クエリ (WP_Query) の最適化
- **現状**: 複数のカスタム投稿タイプが使われており、ループ処理やデータ取得に無駄なクエリが発生している可能性があります。
- **指示の例**:
  > 「`archive-liver.php` と `single-liver.php` の WP_Query の記述をレビューし、データベース負荷を下げるためのキャッシュ対応やパラメータ最適化を提案してください。」

### ③ セキュリティ（サニタイズ・エスケープ）の徹底
- **現状**: ACFで取得した値 (`get_field` など) がそのまま出力されていないか監査が必要です。
- **指示の例**:
  > 「現在のテーマファイル全体を監査し、出力時のエスケープ処理（`esc_html()`, `esc_url()` など）が漏れている箇所をリストアップし、修正してください。」

### ④ SCSS / CSS およびアセットの最適化
- **現状**: Gulpを利用していますが、使われていないCSSや不要なJavaScriptが読み込まれている可能性があります。
- **指示の例**:
  > 「`src/assets/scss/` 配下のファイル構成をレビューし、不要なコードの削除や、Largest Contentful Paint (LCP) を改善するためのCSSロード方法を提案してください。」

### ⑤ エラーログと警告の調査 (PHP Debug)
- **指示の例**:
  > 「`wp-config.php` のデバッグモードを有効にしたと仮定して、一般的なWordPressテーマで発生しやすいPHPのWarningやNoticeを想定し、事前調査を行ってください。」

## 5. 完了した具体的なタスク（実績）

### 実績A：インタビュー機能の完全削除
- トップページの表示セクションの削除、およびカスタム投稿タイプ（`interview`）やタクソノミー（カテゴリー・タグ）の登録解除、テンプレートファイル（`archive-interview.php`等）のクリーンアップを実施しました。

### 実績B：ランキングへの「非公開ライバー」の安全な表示対応
- スプレッドシート連携で「下書き（非表示）」になっているライバーでも、管理画面のランキング設定で選ばれている場合は表示されるように変更しました。また、404エラーを防ぐため、下書き状態のライバーについてはプロフィール詳細ページへのリンク（`<a>`タグ）を自動的に外す対応を実装済みです。

### 実績C：トップページライバースライダーの5名固定
- トップページのスライダーに表示されるライバーを、スプレッドシートの設定に関わらず、ご指定いただいた特定の5名に強制固定するロジックを `front-page.php` に追加しました。
